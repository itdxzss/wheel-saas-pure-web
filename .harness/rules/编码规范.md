# 编码规范

## 基础规则
- Vue 组件必须使用 `<script setup lang="ts">`。
- TypeScript 保持 strict 思维；禁止用 `any` 逃避类型问题，边界数据用 `unknown` 后收窄。
- 默认 `const`，需要重赋值才用 `let`，禁止 `var`。
- 公共函数、API 函数、复杂 composable 必须有明确入参和返回类型。
- 禁止生产代码保留 `console.log`；必要诊断用模板已有消息/日志方式，且上线前清理。

## Vue SFC 顺序
```vue
<script setup lang="ts">
// 1. Vue / Router / Pinia imports
// 2. pure-admin / Element Plus imports
// 3. API / types / utils imports
// 4. props / emits
// 5. route / store / composables
// 6. state
// 7. computed
// 8. methods
// 9. watchers
// 10. lifecycle
</script>

<template>
  <!-- template -->
</template>

<style scoped>
/* scoped style */
</style>
```

## 组件与模板
- 所有 `v-for` 必须有稳定 `:key`，禁止用 index 当 key。
- 禁止在同一元素上同时使用 `v-for` 和 `v-if`。
- 表单必须优先用 `ElForm` / `ElFormItem`，不要手写复杂校验。
- 列表页必须有 loading、empty、error 反馈。
- 频繁切换用 `v-show`，昂贵或少见内容用 `v-if`。
- 禁止直接操作 DOM；确需测量尺寸时使用模板 ref，并清理副作用。

## pure-admin / Element Plus 使用规则
- 表格页优先组合 `RePureTableBar` + `@pureadmin/table` 或 `ElTable`。
- 分页使用 `ElPagination`，不要写自定义分页组件。
- 下拉使用 `ElSelect` / `ElSelectV2` / `ElDropdown`，不要写原生 select 增强层。
- 弹窗使用 `ReDialog` 或 `ElDialog`，抽屉使用 `ElDrawer`。
- 权限展示优先使用 `ReAuth` / `RePerms` 或已有权限指令。
- 图标使用 `ReIcon` / Iconify / Element Plus icons，不手写 SVG 图标系统。

## Composable
- 文件名必须 `useXxx.ts`。
- 只封装可复用逻辑；单页面的一小段简单逻辑不要过度抽象。
- composable 内有事件监听、timer、watch、副作用时必须清理。
- composable 不直接渲染 UI，不返回裸 DOM。
- 接收 ref/reactive 输入时保持响应式，不要只读一次快照。

## API
- API URL 只出现在 `src/api/`。
- 页面禁止 import `axios` 或 `http` 实例。
- API 类型优先来自后端契约或 OpenAPI；临时类型要标明来源并在迁移时收敛。
- 捕获异常不能吞掉；页面必须给出可见错误反馈。
- 禁止在生产 API 文件返回 mock 数据。

## 安全
- 禁止 `v-html`，除非同文件内明确经过可信 sanitizer。
- 动态 `href` / `src` 必须验证协议或来源。
- Token、tenant code 等认证信息只通过统一 auth/http 层读写。
- 不在日志、错误提示、URL 中暴露 token、creds、代理密码、敏感账号凭据。

## 反屎山红线
1. 改代码前先找同类页面和模板示例。
2. 能用 pure-admin / Element Plus 就不自造。
3. 改行为必须删旧路径，不留死代码和注释掉的旧实现。
4. 不为未来想象提前抽象；没有真实复用点不建公共组件。
5. 单次改动只服务一个目标，不顺手重构无关文件。
6. `.vue` 文件超过 600 行不得合入。
7. 禁止“页面里全写完”：API、列定义、校验、批量操作过复杂时必须拆。
