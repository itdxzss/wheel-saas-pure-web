# 开发流程规范

## 标准流程
1. 读 `AGENTS.md` 和相关 `.harness/rules/`。
2. 找同类页面、同类 API、同类路由写法。
3. 大功能创建 `.harness/changes/<feature>/summary.md`。
4. 先写或更新验证用例/验收清单。
5. 实现最小可用改动。
6. 跑 typecheck / lint / build / 相关测试。
7. 记录验证证据。

## 迁移页面流程
每迁移一个旧 SaaS 页面，必须完成：
- 明确旧页面路径和新页面路径。
- 明确菜单 key、route path、module_key、perm_key。
- 明确 API 依赖和后端返回结构。
- 用 pure-admin / Element Plus 组件替代旧手写控件。
- 删除或拒绝迁移旧的自绘控件、全局事件 hack。
- 写明人工点击路径：菜单进入、查询、分页、主要按钮、权限态。

## 变更记录
大于半天的功能、迁移一个完整菜单、改登录/权限/路由/API client，都要建：

```
.harness/changes/<feature-name>/summary.md
```

内容包括目标、影响模块、关键决策、验证结果、遗留风险。

## 验证命令
优先使用项目脚本：
- `pnpm run typecheck`
- `pnpm run lint`
- `pnpm run build`

若本机 pnpm 暂不可用，可临时用 npm 验证，但最终提交必须保持 pnpm lock 一致。

## Review 要求
- 触碰 `.vue` 必须按 Vue reviewer 清单自查。
- 触碰 `.ts` 必须按 TypeScript reviewer 清单自查。
- 触碰路由/权限/登录必须额外检查未授权跳转、403、刷新后动态路由恢复。
- 触碰表格页必须检查 loading、empty、分页、筛选参数、列设置。

## 完成定义
只有满足以下条件才能说完成：
- 代码符合 pure-admin-thin 架构。
- 没有新增自绘基础控件。
- 关键路径已验证。
- 失败项和未验证项已明确说明。
- 变更记录已更新。
