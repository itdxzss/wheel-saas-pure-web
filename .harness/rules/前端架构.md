# 前端架构规范

## 架构原则
本项目以 `pure-admin-thin` 为壳，以 wheel SaaS 业务为内容。UI 能力优先使用模板和 Element Plus，
业务适配层只负责把 wheel 后端的登录、菜单、权限、分页、ApiResponse 映射到模板结构。

## 顶层结构
| 路径 | 责任 |
|------|------|
| `src/layout/` | pure-admin-thin 布局壳；除品牌、导航适配外不做结构性重写 |
| `src/router/` | 静态路由、动态路由、菜单到路由转换、鉴权守卫 |
| `src/store/` | Pinia 全局状态，优先复用模板 store |
| `src/api/` | 只放请求封装和接口函数 |
| `src/views/<domain>/` | 业务页面，按 wheel 业务域纵向组织 |
| `src/views/<domain>/components/` | 只服务该业务域的局部组件 |
| `src/views/<domain>/composables/` | 只服务该业务域的组合逻辑 |
| `src/types/` | 跨域共享类型；优先来自 OpenAPI 或后端契约 |
| `mock/` | 仅开发期 mock；禁止被生产逻辑依赖 |

## 业务域目录
首批业务域固定为：
- `home/` 首页与概览
- `account/` 账号列表、账号分组、账号导入
- `task/` 拉群任务、进群任务、群营销任务
- `group/` 群链接、群组列表、导入链接
- `material/` 素材、模板、头像、昵称
- `resource/` IP、协议、数据包、导出
- `system/` 用户、角色、菜单、审计、租户档案
- `buyer/` 买号上量系统

新增顶层业务域必须有明确菜单归属；不能因为单个页面随手新建顶层目录。

## UI 组件边界
优先级从高到低：
1. pure-admin-thin 自带组件：`RePureTableBar`、`ReDialog`、`RePerms`、`ReAuth`、`ReIcon`。
2. Element Plus：`ElTable` / `ElTableV2`、`ElPagination`、`ElSelect` / `ElSelectV2`、`ElDropdown`、`ElDialog`、`ElDrawer`、`ElForm`。
3. 业务局部组件：只在一个业务域内复用时放 `views/<domain>/components/`。

禁止事项：
- 禁止自绘 select、table、pager、dialog、drawer。
- 禁止全局 DOM 事件拦截器处理通用控件。
- 禁止新增 `components/common` 作为第二套 UI 组件库；确实跨域复用时先确认 pure-admin / Element Plus 是否已有。

## 路由与菜单
路由采用 pure-admin 动态路由格式。wheel 后端菜单字段映射如下：

| wheel 字段 | pure-admin 字段 |
|-----------|-----------------|
| `route_path` | `path` |
| `menu_key` | `name` 或稳定路由名来源 |
| `name` | `meta.title` |
| `icon` | `meta.icon` |
| `module_key` | `meta.module_key` |
| `perm_key` | `meta.perm_key` |
| `view_path` / 前端映射 | `component` |

规则：
- `/api/tenant/me/menus` 是最终菜单来源。
- mock 动态路由只用于开发预览，不得作为生产事实源。
- 所有业务路由必须有 `meta.title`，受控页面必须有 `meta.module_key` 和 `meta.perm_key`。
- 路由组件路径必须落在 `src/views/<domain>/`，不允许从 layout/components 直接挂业务页面。

## API 与数据流
- 页面只调用 `src/api/<domain>.ts` 导出的函数。
- API 文件负责 URL、参数、返回类型，不负责 UI 状态。
- 页面负责 loading、error、empty、分页状态。
- 后端分页统一映射为 `page` / `page_size`，前端分页控件使用 `ElPagination`。
- 后端统一响应按 wheel ApiResponse 处理，不允许页面到处写 `response.data.data` 的重复拆包逻辑；迁移时应收敛到统一 helper。

## 状态管理
- 只放跨页面状态进 Pinia，例如登录用户、租户信息、菜单、权限、主题。
- 页面局部筛选、弹窗、表格选择状态放组件内或 composable。
- 不把大型表格数据长期存 Pinia，避免缓存和权限串租户。

## 页面拆分
- `.vue` 单文件超过 400 行时必须评估拆分。
- `.vue` 单文件超过 600 行不得合入。
- API 请求、表格列定义、表单校验、复杂筛选、批量操作可以拆到同域 composable。
- 子组件只负责展示和局部交互，不直接访问全局 store 或 API，除非它本身就是容器组件。
