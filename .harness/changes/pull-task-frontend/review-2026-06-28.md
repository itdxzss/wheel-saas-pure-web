## Findings

### HIGH 1. 当前 API 指向的 `/api/pull-tasks` 在现有 Armada 后端不存在，页面上线即不可用

新 API 全部走 `armadaRequest` 和 `/api/pull-tasks`：

- `src/api/pull-task.ts:214` `GET /api/pull-tasks`
- `src/api/pull-task.ts:222` `POST /api/pull-tasks`
- `src/api/pull-task.ts:228` 详情
- `src/api/pull-task.ts:232` 群行
- `src/api/pull-task.ts:250` 启动/暂停/停止
- `src/api/pull-task.ts:296` 补充拉手
- `src/api/pull-task.ts:329` 导出

页面挂载后会立即调用列表：

- `src/views/task/pull-task/composables/usePullTaskPage.ts:341`
- `src/views/task/pull-task/composables/usePullTaskPage.ts:751`

但当前 Armada 仓库没有 `PullTaskController` / `/api/pull-tasks` 端点；现役 wheel 老系统契约仍是：

- `../wheel/wheel-saas-web/src/api/pull-task.ts:171` `GET /api/tenant/task-batches`
- `../wheel/wheel-saas-web/src/api/pull-task.ts:230` `POST /api/tenant/task-batches`
- `../wheel/wheel-saas-web/src/api/pull-task.ts:234` 批量 start/pause/resume/stop
- `../wheel/wheel-saas-web/src/api/pull-task.ts:241` 批量删除

影响：`/task/pull` 打开即列表报错，新增、详情、导出、补拉手、批量操作都会失败，除非后端同步落地完整 `/api/pull-tasks` 契约。

建议：要么改回当前真实后端契约并做 snake_case/camelCase 适配，要么在变更 summary 中明确阻塞依赖并不要标为“完成可用”；不能只靠结构 verifier 通过。

### HIGH 2. 自建群是二期项，但新页面可直接选择并提交

一期需求明确自建群/全自动建群是二期项：

- `../wheel/docs/审计-0618/一期需求文档.md:155`
- `../wheel/docs/审计-0618/一期需求文档.md:505`

wheel 老页面隐藏自建群入口，并保留链路供二期放开：

- `../wheel/wheel-saas-web/src/views/pull-task/BatchCreate.vue:310`
- `../wheel/wheel-saas-web/src/views/pull-task/BatchCreate.vue:955`

- `src/views/task/pull-task/components/PullTaskCreateDrawer.vue:48`

- `src/views/task/pull-task/composables/usePullTaskPage.ts:485`

影响：一期未完成的自建群能力从前端漏出，用户可以提交后端未承诺的任务模式。

建议：`CREATE_NEW` 入口一期隐藏或置灰，提交层也要硬拦截并提示“二期功能暂不可用”。

### HIGH 4. “使用管理员”相关必填和禁用规则缺失

一期需求要求：

- 老群链接选择“使用管理员”时必须配置管理员分组。
- 不使用管理员时，不执行管理员授权和后续管理动作。
- 不使用管理员时群组参数不可编辑。

依据：

- `../wheel/docs/审计-0618/一期需求文档.md:160`
- `../wheel/docs/审计-0618/一期需求文档.md:161`
- `../wheel/docs/审计-0618/一期需求文档.md:166`
- `../wheel/docs/审计-0618/一期需求文档.md:172`

老页面也有提交前校验：

- `../wheel/wheel-saas-web/src/views/pull-task/BatchCreate.vue:784`

新实现的问题：

- 默认 `useAdmin: true`，但 `adminGroupId` 没有在 `createTask` 中校验。
  - `src/views/task/pull-task/composables/usePullTaskPage.ts:183`
  - `src/views/task/pull-task/composables/usePullTaskPage.ts:480`
- “群名称 / 群禁言 / 拉人权限 / 群设置权限 / 自动关拉人权限”等群组管理参数没有随 `useAdmin=false` 禁用。
  - `src/views/task/pull-task/components/PullTaskCreateDrawer.vue:298`
  - `src/views/task/pull-task/components/PullTaskCreateDrawer.vue:303`
  - `src/views/task/pull-task/components/PullTaskCreateDrawer.vue:316`
- `adminQuitMode` 在不使用管理员时仍可编辑。
  - `src/views/task/pull-task/components/PullTaskCreateDrawer.vue:289`

影响：可提交“使用管理员但无管理员分组”的无效配置；也可在“不使用管理员”时配置只有管理员才能执行的群管理动作。

建议：提交前校验 `useAdmin && !adminGroupId`；所有管理员依赖字段在 `!useAdmin` 时置灰并不入有效 payload。

### MEDIUM 5. 导出报表后没有显式交单标记

- `../wheel/docs/审计-0618/一期需求文档.md:201`

- `../wheel/wheel-saas-web/src/views/pull-task/BatchList.vue:843`
- `../wheel/wheel-saas-web/src/views/pull-task/BatchList.vue:847`

### MEDIUM 6. 批量删除会拦截 `ENDED`，和 wheel 老系统终态删除口径不一致

新实现只允许 `COMPLETED` 删除：

- `src/views/task/pull-task/composables/usePullTaskPage.ts:607`
- `src/views/task/pull-task/composables/usePullTaskPage.ts:609`

但新状态枚举里有 `ENDED`：

- `src/views/task/pull-task/constants.ts:10`
- `src/views/task/pull-task/constants.ts:12`

wheel 老系统明确把 `COMPLETED` 和 `ENDED` 都视为可删除终态，原因是“手动结束”会置为 `ENDED`：

- `../wheel/wheel-saas-web/src/views/pull-task/BatchList.vue:602`
- `../wheel/wheel-saas-web/src/views/pull-task/BatchList.vue:605`

影响：用户按提示“请手动结束任务后再行删除”把任务结束成 `ENDED` 后，新页面仍然不允许删除。

建议：若沿用现役系统，删除门禁应允许 `COMPLETED` 和 `ENDED`；如果产品坚持“仅已完成”不含 `ENDED`，需要同步修改停止/关闭后的状态语义和提示文案。

### LOW 9. Harness 编码规范：composable 中直接操作 DOM

编码规范禁止直接操作 DOM：

- `.harness/rules/编码规范.md:40`

- `src/views/task/pull-task/composables/usePullTaskPage.ts:264`
- `src/views/task/pull-task/composables/usePullTaskPage.ts:267`

影响：这是 harness 明确禁止项，且导出下载逻辑混在 800 行 composable 里，后续复用和测试都不理想。

建议：抽到项目已有下载工具或封装一个局部 `downloadTextFile` utility；如必须 DOM 下载，至少隔离在工具层并补注释说明。

### LOW 10. SFC 超过 400 行，需要拆分评估记录

Harness 页面拆分规则：`.vue` 超过 400 行必须评估拆分，超过 600 行不得合入。

当前：

- `src/views/task/pull-task/index.vue` 454 行
- `src/views/task/pull-task/components/PullTaskCreateDrawer.vue` 416 行
- `src/views/task/pull-task/components/PullTaskDetailDrawer.vue` 380 行
- `src/views/task/pull-task/composables/usePullTaskPage.ts` 807 行

影响：没有超过 600 行硬线，但 `index.vue` 和创建抽屉都超过 400 行，且 composable 已经承担列表、创建、详情、导出、补拉手、批量操作所有状态，维护成本偏高。

建议：至少把主表格拆成 `PullTaskTable.vue`，创建抽屉按 tab 拆局部子组件，composable 拆成 list/create/detail/export 几个内部 composable。
